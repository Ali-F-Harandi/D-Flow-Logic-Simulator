[Beta-1] – 2026-05-06 (hotfix)
    Fixed: Canvas.js constructor now creates #canvas-container if it doesn't exist, preventing "appendChild of null" error.
    Root cause identified: main.js was calling canvas.addComponent(comp) directly AND emitting
    an event that also triggered canvas.addComponent. This created two DOM elements per component,
    causing a static "ghost" gate and wiring to connect to the wrong element.
    Fixed by removing the direct canvas.addComponent call from main.js; the Canvas now solely
    handles rendering via its component-created listener.
    Also added dragstart prevention on canvas components to suppress any native drag images.
    Dragging now works perfectly, and wiring connects to the correct input/output dots.
    - Fixed wire update after component drag: _updateWiresForComponent now correctly uses node ID prefixes.
    - Added component deletion (right-click on component).
    - Added wire deletion (right-click on wire path).
    - Engine.js: added removeComponent(compId) and disconnect(wireId) methods.
    Wires now visually follow their connected components when dragged.
    Known issues: none critical; undo/redo not yet implemented.
    Engine.connect now returns the engine's internal wire ID.
    Canvas._completeConnection stores that ID as wire.engineId.
    Canvas._deleteWire uses engine.engineId to call engine.disconnect, correctly removing the logical connection.
    Wire deletion now fully functional, both visual and logical.
